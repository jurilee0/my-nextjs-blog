안녕하세요. 첫 포스팅은 초보자를 위한 Git 가이드를 준비했습니다. 천천히 함께 살펴봅시다! 😊

## 1️⃣ Git이 뭔가요?

여러분도 이런 경험 있으신가요?

- "최종.txt", "진짜최종.txt", "진짜최최종.txt" 파일을 만들어본 적이 있다
- 동료와 같은 파일을 수정하다가 서로 작업 내용이 엉킨 것 같은데?
- 이전에 작성했던 코드로 돌아가고 싶은데 방법을 모르겠다

이런 문제들을 해결해주는 도구가 바로 Git입니다!

### **Git이란?**

Git은 여러분의 작업 내용을 안전하게 저장하고 관리해주는 '타임머신' 같은 도구예요.
마치 게임에서 세이브 포인트를 만들듯 작업 중간중간 저장(커밋)을 해두면 나중에 그 시점으로 돌아갈 수 있답니다.

## 2️⃣ Git 설치하기

#### Windows 사용자라면?

1. [Git 공식 사이트](https://git-scm.com)에 접속하세요
2. 다운로드 버튼을 클릭하세요
3. 설치 파일을 실행하세요
4. 설치 과정에서 'Git Bash'를 선택하거나 따로 설치해주시는 것을 권장해요

#### Mac 사용자라면?

터미널을 열고 아래 명령어를 입력하세요

```bash
brew install git
```

설치가 잘 되었는지 터미널(또는 Git Bash)을 열고 입력해보세요.

```bash
git --version
# git version 2.34.1 이런 식으로 나오면 성공입니다!
```

## 3️⃣ Git 시작하기: 첫 프로젝트 만들기

### **Git에게 나를 소개하기**

Git을 처음 사용한다면, 먼저 자신을 소개해줍시다.

```bash
# 이름 설정하기
git config --global user.name "홍길동"

# 이메일 설정하기
git config --global user.email "gildong@example.com"
```

### **첫 번째 Git 프로젝트 시작하기**

1. **프로젝트 폴더 만들기**

```bash
# 바탕화면에 my-project 폴더 만들기
cd Desktop
mkdir my-project
cd my-project
```

2. **Git 시작 선언하기**

```bash
git init
# 'Git 저장소를 시작할게요!' 라고 선언하는 거예요
```

3. **첫 파일 만들기**

```bash
# README.md 파일을 만들어봅시다
echo "# 나의 첫 프로젝트" > README.md
```

4. **변경사항 저장하기 (커밋하기)**

```bash
# 1) 파일을 저장소에 추가하기
git add README.md

# 2) 변경사항을 기록하기
git commit -m "첫 번째 커밋입니다!"
```

### **작업 상태 확인하기**

```bash
# 현재 상태 보기
git status

# 작업 기록 보기
git log
```

## 4️⃣ Git으로 코드 관리하기

### **기본 작업 흐름**

1. **파일 수정하기**

   - 평소처럼 파일을 수정하세요

2. **변경사항 확인하기**

```bash
git status
# 빨간색으로 표시된 파일이 변경된 파일이에요
```

3. **변경사항 스테이징하기**

```bash
git add 파일이름
# 또는 모든 변경사항을 한번에
git add .
```

4. **변경사항 저장하기 (커밋)**

```bash
git commit -m "어떤 것을 변경했는지 설명을 적어주세요"
```

## 5️⃣ Branch: 작업을 나누어 관리하기

### **브랜치가 뭔가요?**

독립적인 작업 공간이에요. 마치 게임에서 세이브 파일을 여러 개 만드는 것처럼,
각각 다른 작업을 동시에 진행할 수 있답니다.

### **브랜치 사용해보기**

1. **브랜치 목록 보기**

```bash
git branch
# * main <- 현재 있는 브랜치를 보여줘요
```

2. **새 브랜치 만들기**

```bash
# login 기능을 만들 새로운 브랜치를 생성한다면 이런 이름을 붙여줘요
git branch feature/login

# 브랜치를 만들고 바로 이동하고 싶다면
git checkout -b feature/login
```

3. **브랜치 이동하기**

```bash
# login 브랜치로 이동
git checkout feature/login

# 다시 main 브랜치로 돌아가기
git checkout main
```

4. **브랜치 합치기 (병합)**

```bash
# 1) main 브랜치로 이동
git checkout main

# 2) login 브랜치의 내용을 main에 가져오기
git merge feature/login
```

## 6️⃣ GitHub: 내 코드를 온라인에 저장하기

### **GitHub가 뭔가요?**

GitHub는 Git으로 관리하는 코드를 온라인에 저장하고 공유할 수 있는 곳이에요.
마치 구글 드라이브처럼 파일을 저장하고 공유할 수 있답니다!

### **GitHub 시작하기**

1. **GitHub 계정 만들기**

   - [GitHub](https://github.com)에 접속해서 회원가입하세요
   - 이메일 인증을 꼭 완료해주세요

2. **내 프로젝트를 GitHub에 올리기**

```bash
# 1) GitHub와 연결하기
git remote add origin https://github.com/나의아이디/저장소이름.git

# 2) 내 코드를 GitHub에 올리기
git push -u origin main
```

3. **GitHub의 프로젝트 가져오기**

```bash
# 다른 사람의 프로젝트 복사해오기
git clone https://github.com/다른사람아이디/저장소이름.git
```

### **GitHub에서 협업하기**

1. **변경사항 가져오기**

```bash
# GitHub의 최신 변경사항 가져오기
git pull origin main
```

2. **내 변경사항 올리기**

```bash
# 1) 변경사항 확인
git status

# 2) 변경사항 스테이징
git add .

# 3) 변경사항 커밋
git commit -m "로그인 페이지 디자인 수정"

# 4) GitHub에 올리기
git push origin main
```

## 7️⃣ 자주 발생하는 문제 해결하기

### **충돌이 발생했어요!**

충돌은 같은 파일의 같은 부분을 여러 사람이 수정했을 때 발생해요.
아래 단계를 따라 해결할 수 있어요

1. **충돌 확인하기**

```bash
git status
# 빨간색으로 충돌난 파일이 보여요
```

2. **충돌 파일 열어보기**

```
<<<<<<< HEAD
여기는 내가 수정한 내용
=======
여기는 다른 사람이 수정한 내용
>>>>>>> feature/login
```

3. **충돌 해결하기**

   - 충돌난 부분을 열어서 필요한 내용만 남기세요
   - `<<<<<<<`, `=======`, `>>>>>>>`는 모두 지워주세요

4. **해결 완료하기**

```bash
git add .
git commit -m "충돌 해결: 로그인 페이지 병합"
```

## 8️⃣ Git 핵심 개념 이해하기

### **Git의 4가지 파일 상태**

Git은 파일을 4가지 상태로 관리해요

1. **Untracked (추적되지 않음)**

   - Git이 아직 관리하지 않는 파일이에요
   - 새로 만든 파일이 이 상태예요

2. **Unmodified (수정되지 않음)**

   - Git이 관리하는 파일이지만 변경사항이 없어요
   - 마지막 커밋 이후 수정하지 않은 파일이에요

3. **Modified (수정됨)**

   - Git이 관리하는 파일이 수정된 상태예요
   - 아직 스테이징되지 않은 변경사항이에요

4. **Staged (스테이징됨)**
   - 다음 커밋에 포함될 변경사항이에요
   - `git add` 명령어로 이 상태가 돼요

### **Git 명령어 상세 **

#### **기본 명령어**

```bash
# 저장소 관련
git init              # 새 저장소 만들기
git clone <url>       # 원격 저장소 복제하기
git remote -v         # 연결된 원격 저장소 보기
git remote add origin URL  # 원격 저장소 추가
git push origin 브랜치명   # 변경사항 원격 저장소에 푸시
git pull origin 브랜치명   # 원격 저장소의 변경사항 가져오기
git fetch                  # 원격 변경사항 조회

# 변경사항 관리
git status           # 현재 상태 보기
git diff             # 변경사항 자세히 보기
git diff --staged    # 스테이징된 변경사항 보기

# 커밋 관련
git add 파일명         # 변경 파일 스테이징
git commit -m "메시지"  # 변경사항 커밋
git commit --amend     # 마지막 커밋 메시지 수정
git reset HEAD 파일명  # 스테이징 취소
git log              # 커밋 이력 보기
git log --oneline    # 커밋 이력 한 줄로 보기
git log --graph      # 브랜치 그래프와 함께 보기

# 브랜치 관련
git branch             # 로컬 브랜치 목록
git branch -r          # 원격 브랜치 목록
git checkout 브랜치명  # 브랜치 이동
git checkout -b 새브랜치  # 새 브랜치 생성 및 이동
git merge 브랜치명     # 브랜치 병합
git branch -d 브랜치명  # 브랜치 삭제

# Stash 작업 (임시 저장)
git stash                # 변경사항 임시 저장
git stash list          # 임시 저장 목록 보기
git stash pop           # 가장 최근 stash 적용
git stash drop          # 가장 최근 stash 삭제

# Reset & Revert
git reset --soft HEAD^   # 최근 커밋 취소 (변경사항 유지)
git reset --hard HEAD^   # 최근 커밋 완전히 삭제
git revert <커밋해시>    # 특정 커밋 되돌리기
```

## 9️⃣ 실전 협업 전략

### **Git Flow 전략**

대규모 프로젝트에서 많이 사용하는 브랜치 전략이에요. 아래처럼 용도 별 브랜치 나누어 사용해요.

```bash
# 주요 브랜치
main     # 제품 출시 브랜치
develop  # 개발 브랜치

# 보조 브랜치
feature  # 기능 개발
release  # 출시 준비
hotfix   # 긴급 수정
```

### **GitHub Flow 전략**

작은 팀이나 지속적 배포를 하는 프로젝트에 적합해요.

- **기본 규칙**
  - main 브랜치는 항상 배포 가능한 상태로 유지
  - 새로운 작업은 브랜치를 만들어서 진행
  - Pull Request로 코드 리뷰 진행
  - 리뷰가 승인되면 main에 병합

```bash
# GitHub Flow 예시
git checkout -b feature/login  # 새 기능 브랜치 생성

# ... 작업 진행 ...

git push origin feature/login  # 원격 저장소에 올리기
# Pull Request 생성 및 리뷰 진행
```

### **효과적인 커밋 메시지 작성법**

```bash
# 아래처럼 특정 프리픽스를 붙이는 컨벤션을 주로 사용해요
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서 수정
style: 코드 포맷팅
refactor: 코드 리팩토링
test: 테스트 코드
chore: 기타 변경사항

# 커밋 메시지 예시
git commit -m "feat: 로그인 기능 구현

- 이메일/비밀번호 유효성 검사 추가
- 로그인 실패 시 에러 메시지 표시
- 로그인 성공 시 메인 페이지로 이동"

```

### **Merge 방법론**

1. **Fast-Forward Merge**

   - 브랜치가 서로 달라진 내용이 없을 때 직선적으로 병합됩니다.

```bash
git merge 브랜치명
```

2. **Recursive Merge**
   - 두 브랜치의 변경 사항을 합쳐 새로운 커밋을 만듭니다.

```bash
git merge 브랜치명
```

3. **Squash Merge**
   여러 커밋을 하나로 합쳐 병합해요. 히스토리를 간결하게 유지할 수 있습니다.

```bash
git merge --squash 브랜치명
```
