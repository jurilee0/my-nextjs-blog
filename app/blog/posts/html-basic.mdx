## 1️⃣ HTML이란?

HTML은 'Hypertext Markup Language'의 약자로 웹 문서의 구조를 정의하는 언어입니다. 웹 페이지를 만들 때 가장 먼저 배우게 되는 언어로, HTML을 통해 웹 페이지의 기본적인 틀을 만들 수 있습니다.

### **HTML의 기본 구조**

HTML 문서는 용도에 맞는 여러 `<tag>`들로 구성합니다.
예를 들어,`<p>` 글 문단을 나타내며 `<img>` 태그는 이미지를 삽입하는 데 사용합니다.
태그들은 부모/자식, 형제 관계를 맺으며 속성(attr)과 값(value)을 통해 요소를 정의할 수 있습니다.

## 2️⃣ HTML 문서의 기본 구성

HTML 문서는 다음과 같은 기본 구조를 가집니다:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>나의 첫 웹 페이지</title>
  </head>
  <body>
    <h1>안녕하세요!</h1>
    <p>이것은 나의 첫 HTML 문서입니다.</p>
  </body>
</html>
```

- `<!DOCTYPE html>`: HTML5 문서임을 선언합니다.
- `<html>`: HTML 문서의 루트 요소입니다.
- `<head>`: 메타데이터를 포함하는 부분으로, 문서의 제목, 스타일, 스크립트 등을 정의합니다.
- `<title>`: 문서의 제목을 정의합니다.
- `<body>`: 실제 콘텐츠가 들어가는 부분입니다.

## 3️⃣ HTML 요소와 속성

### **HTML 요소**

태그는 기본적으로 시작 태그와 종료 태그로 구성되지만 단독 태그도 있어요.

### **HTML 속성**

HTML 요소는 속성을 가질 수 있으며, 속성은 요소에 대한 추가 정보를 제공해요.
아래 `<img>` 태그는 `src` 속성을 사용하여 이미지의 경로를 지정하고 `alt` 속성을 사용하여 이미지에 대한 대체 텍스트를 제공합니다.

```html
<img src="image.jpg" alt="예시 이미지" />
```

## 4️⃣ HTML의 다양한 요소

### **텍스트 요소**

- `<h1> ~ <h6>`: 제목을 나타내며, `<h1>`이 가장 중요합니다. `<h1>`은 가장 큰 제목, `<h6>`은 가장 작은 제목을 나타냅니다.
- `<p>`: 본문 단락을 나타냅니다. 문단을 구분할 때 사용합니다.
- `<strong>`: 강조를 나타내며, `<b>`보다 의미가 있습니다. 중요한 내용을 강조할 때 사용합니다.
- `<em>`: 이탤릭체를 나타내며, `<i>`보다 의미가 있습니다. 강조하고 싶은 부분을 기울임체로 표시합니다.
- `<pre>`: 작성한 그대로 화면에 노출되며, 주로 코드나 서식을 유지할 때 사용합니다.
- `<br>`: 줄 바꿈을 나타냅니다. 문단 내에서 줄을 바꿀 때 사용합니다.
- `<del>`: 삭제된 텍스트를 나타냅니다. 수정 전의 내용을 표시할 때 사용합니다.
- `<ins>`: 삽입된 텍스트를 나타냅니다. 새로 추가된 내용을 강조할 때 사용합니다.
- `<sub>`: 아래 첨자를 나타냅니다. 화학식이나 수학식에서 아래 첨자를 표시할 때 사용합니다.
- `<sup>`: 윗첨자를 나타냅니다. 제곱이나 기호를 표시할 때 사용합니다.

### **링크와 미디어**

- `<a>`: 다른 문서나 페이지로 이동하는 링크를 만듭니다. `href` 속성을 사용하여 링크의 목적지를 지정합니다.
- `<img>`: 이미지를 삽입합니다. `src` 속성을 사용하여 이미지의 경로를 지정하고, `alt` 속성을 사용하여 대체 텍스트를 제공합니다.
- `<audio>`: 음성을 삽입합니다. `src` 속성을 사용하��� 오디오 파일의 경로를 지정합니다.
- `<video>`: 영상을 삽입합니다. `src` 속성을 사용하여 비디오 파일의 경로를 지정합니다.
- `<source>`: `<audio>`나 `<video>` 요소 내에서 여러 미디어 소스를 지정할 때 사용합니다.
- `<figure>`: 이미지나 도표 등 컨텐츠를 묶는 요소입니다. `<figcaption>`과 함께 사용하여 설명을 제공합니다.
- `<figcaption>`: `figure`의 설명을 제공합니다.
- `<iframe>`: 다른 문서를 현재 페이지에 삽입합니다. 주로 외부 콘텐츠를 포함할 때 사용합니다.
- `<canvas>`: 그래픽을 그릴 수 있는 공간을 제공합니다. 자바스크립트를 사용하여 그래픽을 그릴 수 있습니다.

### **목록**

- `<ul>`: 순서가 없는 목록을 만듭니다. `<li>` 요소와 함께 사용하여 목록 항목을 정의합니다.
- `<ol>`: 순서가 있는 목록을 만듭니다. 마찬가지로 `<li>` 요소와 함께 사용하여 목록 항목을 정의합니다.
- `<li>`: 목록 항목을 정의합니다. `<ul>` 또는 `<ol>` 내에서만 사용합니다.
- `<dl>`: 정의형 목록을 만듭니다. `<dt>`와 `<dd>` 요소와 함께 사용하여 용어와 정의를 나열합니다.
- `<dt>`: 정의형 목록에서 용어를 나타냅니다.
- `<dd>`: 정의형 목록에서 용어의 정의를 나���냅니다.

### **폼 요소**

- `<form>`: 사용자 입력을 받을 수 있는 폼 영역을 만듭니다. `action` 속성을 사용하여 폼 데이터를 제출할 URL을 지정합니다.
- `<input>`: 다양한 유형을 입력 받을 수 있는 요소입니다. `type` 속성을 사용하여 입력 유형을 지정합니다.
  - `type="text"`: 일반 텍스트 입력 필드
  - `type="password"`: 입력된 텍스트 마스킹
  - `type="email"`: 이메일 주소를 입력받는 필드
  - `type="url"`: URL 입력 받는 필드
  - `type="number"`: 숫자만 입력 받는 필드
  - `type="tel"`: 전화번호를 입력 받는 필드
  - `type="date"`: 날짜를 입력 받는 필드
  - `type="time"`: 시간을 입력 받는 필드
  - `type="checkbox"`: 체크 박스를 생성
  - `type="radio"`: 라디오 버튼 생성, 그룹 내 하나만 선택 가능
  - `type="file"`: 파일을 업로드하는 필드
  - `type="hidden"`: 화면에 표시되지 않는 숨겨진 입력 필드
  - `type="submit"`: 폼 데이터를 서버로 제출
  - `type="reset"`: 폼의 모든 입력 필드를 초기화
  - `type="range"`: 지정된 범위를 입력 받는 필드
  - `type="search"`: 검색 입력 필드
  - `type="color"`: 색상을 선택하는 입력 필드(브라우저 제공)
  - `type="week"`: �� 단위로 날짜를 입력받는 필드
  - `type="month"`: 월 단위로 날짜를 입력받는 필드
  - `type="datetime-local"`: 날짜와 시간을 함께 입력받는 필드
- `<textarea>`: 여러 줄의 텍스트를 입력 받는 필드입니다.
- `<select>`: 드롭 다운 목록으로 입력 받는 필드입니다. `<option>` 태그와 함께 사용하여 선택 항목을 정의합니다.
- `<option>`: `<select>` 요소 내에서 선택 항목을 정의합니다.
- `<button>`: 버튼을 생성하고 사용자 반응을 유도하거나 폼을 제출 할 때 사용합니다.

### **테이블**

- `<table>`: 데이터를 행과 열로 구성합니다.
- `<caption>`: 테이블의 제목을 지정합니다.
- `<thead>`: 테이블의 머리글을 그룹화합니다.
- `<tbody>`: 테이블의 본문을 그룹화합니다.
- `<tfoot>`: 테이블의 바닥글을 그룹화합니다.
- `<tr>`: 테이블의 행(row)을 정의합니다.
- `<th>`: 테이블의 제목 셀을 정의합니다. 주로 머리글에 사용됩니다.
- `<td>`: 테이블의 일반 셀을 정의하는 태그 기본 단위입니다.
- `<colgroup>`: 열을 그룹화할 때 사용합니다.
- `<col>`: `<colgroup>`에서 개별 열에 스타일이나 속성을 적용합니다.

### **기타 요소**

- `<address>`: 연락처, 주소 정보를 나타냅니다.
- `<blockquote>`: 인용문을 삽입합니다.
- `<hr>`: 수평선을 삽입하여 콘텐츠를 구분합니다.
- `<mark>`: 강조 표시를 합니다.
- `<code>`: 코드 블록을 표시합니다.
- `<details>`: 숨길 수 있거나 표시할 수 있는 추가 세부 사항을 제공합니다.
- `<summary>`: `<details>`의 요약 부분을 나타냅니다.
- `<time>`: 날짜나 시간을 표현합니다.
- `<output>`: 계산 된 값이나 사용자 입력에 의해 변화하는 결과 값을 나타냅니다.
- `<progress>`: 진행률을 표시합니다.
- `<meter>`: 사용자가 정의한 범위 내에서 특정 값의 비율을 나타냅니다.

### **레이아웃 요소**

- `<div>`: 레이아웃을 위해 태그들을 나누거나 묶을 때 사용하는 대표적인 블럭 요소입니다.
- `<span>`: 대표적인 인라인 요소로 텍스트의 일부를 스타일링할 때 사용합니다.

### **시맨틱 태그**

시맨틱 태그는 "의미를 가진 태그"로 웹 페이지의 구조를 더 명확하게 표현합니다!
의미를 가진 태그임을 쉽게 유추한다면 검색 엔진 최적화(SEO)에 도움이 되겠죠?
또 웹 접근성이 향상되며 코드의 가독성이 좋아진답니다.

- 시멘틱 태그의 종류
  - `<header>`: 페이지나 섹션의 헤더 영역을 나타냅니다.
  - `<nav>`: 네비게이션 메뉴를 정의합니다.
  - `<main>`: 페이지의 주요 콘텐츠를 나타냅니다.
  - `<article>`: 독립적인 콘텐츠를 나타냅니다.
  - `<section>`: 관련된 콘텐츠를 그룹화합니다.
  - `<aside>`: 페이지의 부가적인 콘텐츠를 나타냅니다.
  - `<footer>`: 페이지나 섹션의 푸터 영역을 나타냅니다.

### **HTML 엔티티 코드**

엔티티코드는 HTML태그와 혼동 될 수 있는 특수 문자를 표현하기 위해 사용됩니다.
가볍게 알아두면 좋을 것 같아요!

- `&lt;`: `<`
- `&gt;`: `>`
- `&amp;`: `&`
- `&quot;`: `"`
- `&apos;`: `'`
- `&copy;`: `©`
- `&reg;`: `®`
- `&trade;`: `™`
- `&nbsp;`: `공백`

## 6️⃣ 웹 표준(Web Standard)

웹 표준이란, 넓게 웹에서 사용되는 표준 기술이나 규칙을 말합니다. W3C 권고안과 가이드라인을 준수하며
표준 기술을 바탕으로 하기 때문에 다양한 브라우저에서 호환성을 보장합니다.

## 7️⃣ 웹 접근성

신체적/환경적 제약이 있는 사용자를 포함한 모든 사용자들이 동등하게 접근하고 이용할 수 있는 것을 목표로 합니다.

특히 ARIA라는 접근성을 향상 시키기 위한 기술이 있으니 꼭 한번 살펴보세요! `role, aria-label, aria-live, aria-expanded`

## 8️⃣ OG

오픈 그래프는 웹 문서에 대한 메타데이터를 제공하는 프로토콜을 말합니다.쉽게 생각하면 소셜 미디어를 통해 페이지가 공유 될 때 어떻게 표시되는지 생각해보면 아래 요소를 포함 한 것을 확인 할 수 있어요!

- `og:title`: 페이지 제목
- `og:description`: 페이지의 간략한 설명
- `og:image`: 페이지를 대표하는 이미지 URL
- `og:url`: 페이지 URL
- `og:type`: 페이지의 유형 - `website`, `article`, `video` 등

## 9️⃣ UTF-8

UTF-8은 유니코드 문자 인코딩 방식 중 하나로, 전 세계의 다양한 문자와 기호를 표현할 수 있습니다.
웹에서 가장 널리 사용되고 있는 문자 인코딩 방식이에요.

- UTF-8은 아스키(ASCII)와 호환되며, 영어뿐만 아니라 한글, 중국어, 아랍어 등 다양한 언어를 지원합니다.
- UTF-8은 문자에 따라 가변 길이 인코딩을 사용하여, 영어와 같은 아스키 문자는 1바이트로, 다른 문자는 2바이트 이상으로 저장합니다.
